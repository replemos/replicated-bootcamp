# EC v3 Testing Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a manual EC v3 testing guide and a CI e2e workflow that installs playball.exe via Embedded Cluster v3 headlessly on a CMX VM, verified by clean exit code.

**Architecture:** Release creation moves to a dedicated `release` job in `build-test.yml`; `e2e` and `e2e-ec3` run in parallel against the shared release, each creating their own customer. The new `e2e-ec3.yml` creates a CMX VM, SCPs config-values, SSHes in to download+run the EC installer, and verifies clean exit. Channel archival happens in `build-test.yml` after both e2e jobs complete.

**Tech Stack:** GitHub Actions, Replicated CLI, `replicatedhq/replicated-actions`, EC v3 installer, CMX VMs

`docs/testing-ec3.md`:
```markdown
# Testing with Embedded Cluster v3

Use the [Replicated Compatibility Matrix (CMX)](https://docs.replicated.com/vendor/testing-about) to spin up an on-demand Ubuntu VM and test EC v3 installation end-to-end.

## Prerequisites

- `replicated` CLI installed and authenticated
- `REPLICATED_API_TOKEN` set in your environment
- `helm` installed
- SSH key at `.ssh/id_ed25519` (see below)

## Generate SSH Key

VMs require an SSH key. Generate one if not present:

```bash
mkdir -p .ssh
ssh-keygen -t ed25519 -f .ssh/id_ed25519 -N "" -C "ethan@cmx"
```

The private key is gitignored. The public key is committed.

## Create a Release

Package the chart and create a Replicated release:

```bash
HELM_VERSION="0.1.0-ec3-test"
helm dependency update deploy/charts
helm package deploy/charts -d deploy/manifests --version "$HELM_VERSION"

replicated release create \
  --yaml-dir deploy/manifests \
  --version "$HELM_VERSION" \
  --promote Unstable \
  --token $REPLICATED_API_TOKEN \
  --app playball-exe

rm deploy/manifests/*.tgz
```

Note the channel slug (e.g. `unstable`) from the output.

## Create a Customer

```bash
replicated customer create \
  --name "ec3-test" \
  --channel Unstable \
  --type dev \
  --embedded-cluster-download \
  --token $REPLICATED_API_TOKEN \
  --app playball-exe
```

Note the license ID from the output.

## Create a VM

```bash
replicated vm create \
  --distribution ubuntu \
  --version 24.04 \
  --ttl 2h \
  --ssh-public-key .ssh/id_ed25519.pub \
  --token $REPLICATED_API_TOKEN
```

Wait for `STATUS` to show `running`:

```bash
replicated vm ls --token $REPLICATED_API_TOKEN
```

Note the VM ID.

## Create config-values.yaml

```bash
cat > /tmp/config-values.yaml <<'EOF'
apiVersion: kots.io/v1beta1
kind: ConfigValues
spec:
  values:
    postgresql_enabled:
      value: "1"
    redis_enabled:
      value: "1"
EOF
```

## SCP config-values.yaml to VM

```bash
USERNAME=$(cut -d' ' -f3 .ssh/id_ed25519.pub | cut -d'@' -f1)
SCP_ENDPOINT=$(replicated vm scp-endpoint <vm-id> --username "$USERNAME" --token $REPLICATED_API_TOKEN)
SCP_HOST=$(echo "$SCP_ENDPOINT" | sed 's|ssh://[^@]*@||' | cut -d: -f1)
SCP_PORT=$(echo "$SCP_ENDPOINT" | cut -d: -f3)

scp -i .ssh/id_ed25519 \
  -P "$SCP_PORT" \
  -o StrictHostKeyChecking=no \
  -o UserKnownHostsFile=/dev/null \
  /tmp/config-values.yaml \
  "${USERNAME}@${SCP_HOST}:/tmp/config-values.yaml"
```

## SSH and Install

```bash
SSH_ENDPOINT=$(replicated vm ssh-endpoint <vm-id> --username "$USERNAME" --token $REPLICATED_API_TOKEN)
SSH_HOST=$(echo "$SSH_ENDPOINT" | sed 's|ssh://[^@]*@||' | cut -d: -f1)
SSH_PORT=$(echo "$SSH_ENDPOINT" | cut -d: -f3)

ssh -i .ssh/id_ed25519 \
  -p "$SSH_PORT" \
  -o StrictHostKeyChecking=no \
  -o UserKnownHostsFile=/dev/null \
  "${USERNAME}@${SSH_HOST}"
```

On the VM, download and run the installer (the license is embedded in the tgz):

```bash
# Replace <channel-slug>, <version>, and <license-id> with your values
curl -f "https://replicated.app/embedded/playball-exe/<channel-slug>/<version>" \
  -H "Authorization: <license-id>" \
  -o playball-exe.tgz

tar xzf playball-exe.tgz

sudo ./playball-exe install \
  --license license.yaml \
  --headless \
  --yes \
  --installer-password "yourpassword" \
  --config-values /tmp/config-values.yaml
```

## Access Admin Console

After install completes, expose port 30080:

```bash
replicated vm port expose <vm-id> --port 30080 --protocol http,https --token $REPLICATED_API_TOKEN
```

The command returns a public URL. Open it in a browser to access the Admin Console.

## Clean Up

Always delete the VM when done:

```bash
replicated vm rm <vm-id> --token $REPLICATED_API_TOKEN
```

- [ ] **Step 1: Create the workflow file**

`.github/workflows/e2e-ec3.yml`:
```yaml
name: E2E Test EC v3 on CMX VM

on:
  workflow_call:
    inputs:
      image-tag:
        description: Docker image tag to deploy
        required: true
        type: string
      pr-number:
        description: Pull request number
        required: true
        type: string
      channel-slug:
        description: Replicated channel slug
        required: true
        type: string
      version:
        description: Release version label
        required: true
        type: string
  workflow_dispatch:
    inputs:
      image-tag:
        description: Docker image tag to deploy
        required: true
        type: string
      pr-number:
        description: Pull request number
        required: false
        type: string
        default: 'manual'
      channel-slug:
        description: Replicated channel slug
        required: true
        type: string
      version:
        description: Release version label
        required: true
        type: string

jobs:
  e2e-ec3:
    runs-on: ubuntu-latest
    timeout-minutes: 30
    steps:
      - uses: actions/checkout@v4

      - name: Install Replicated CLI
        run: |
          version=$(curl -s https://api.github.com/repos/replicatedhq/replicated/releases/latest \
            | grep -m1 -Po '"tag_name":\s*"v\K[^"]+')
          curl -Ls "https://github.com/replicatedhq/replicated/releases/download/v${version}/replicated_${version}_linux_amd64.tar.gz" \
            -o replicated.tar.gz
          tar xf replicated.tar.gz replicated && rm replicated.tar.gz
          sudo mv replicated /usr/local/bin/replicated

      - name: Create test customer
        id: create-customer
        uses: replicatedhq/replicated-actions/create-customer@v1
        with:
          app-slug: playball-exe
          api-token: ${{ secrets.REPLICATED_API_TOKEN }}
          customer-name: ci-ec3-pr-${{ inputs.pr-number }}-${{ github.run_number }}
          license-type: dev
          channel-slug: ${{ inputs.channel-slug }}
          is-kots-install-enabled: "false"
          is-embedded-cluster-download-enabled: "true"

      - name: Generate SSH keypair
        run: |
          mkdir -p .ssh
          ssh-keygen -t ed25519 -f .ssh/id_ed25519 -N "" -C "ci@cmx"

      - name: Generate installer password
        run: echo "INSTALLER_PASSWORD=$(openssl rand -hex 16)" >> $GITHUB_ENV

      - name: Create CMX VM
        id: create-vm
        run: |
          VM_ID=$(replicated vm create \
            --distribution ubuntu \
            --version 24.04 \
            --ttl 1h \
            --ssh-public-key .ssh/id_ed25519.pub \
            --token ${{ secrets.REPLICATED_API_TOKEN }} \
            | awk 'NR==2 {print $1}')
          echo "vm-id=$VM_ID" >> $GITHUB_OUTPUT

      - name: Wait for VM to be running
        run: |
          VM_ID="${{ steps.create-vm.outputs.vm-id }}"
          while true; do
            STATUS=$(replicated vm ls --token ${{ secrets.REPLICATED_API_TOKEN }} \
              | awk -v id="$VM_ID" '$1==id {print $7}')
            echo "VM status: $STATUS"
            [ "$STATUS" = "running" ] && break
            sleep 10
          done

      - name: Write config-values.yaml
        run: |
          cat > /tmp/config-values.yaml <<'EOF'
          apiVersion: kots.io/v1beta1
          kind: ConfigValues
          spec:
            values:
              postgresql_enabled:
                value: "1"
              redis_enabled:
                value: "1"
          EOF

      - name: SCP config-values.yaml to VM
        run: |
          VM_ID="${{ steps.create-vm.outputs.vm-id }}"
          SCP_ENDPOINT=$(replicated vm scp-endpoint "$VM_ID" \
            --username ci \
            --token ${{ secrets.REPLICATED_API_TOKEN }})
          SCP_HOST=$(echo "$SCP_ENDPOINT" | sed 's|ssh://[^@]*@||' | cut -d: -f1)
          SCP_PORT=$(echo "$SCP_ENDPOINT" | cut -d: -f3)
          scp -i .ssh/id_ed25519 \
            -P "$SCP_PORT" \
            -o StrictHostKeyChecking=no \
            -o UserKnownHostsFile=/dev/null \
            /tmp/config-values.yaml \
            "ci@${SCP_HOST}:/tmp/config-values.yaml"

      - name: Download and install EC v3
        env:
          LICENSE_ID: ${{ steps.create-customer.outputs.license-id }}
          CHANNEL: ${{ inputs.channel-slug }}
          VERSION: ${{ inputs.version }}
        run: |
          VM_ID="${{ steps.create-vm.outputs.vm-id }}"
          SSH_ENDPOINT=$(replicated vm ssh-endpoint "$VM_ID" \
            --username ci \
            --token ${{ secrets.REPLICATED_API_TOKEN }})
          SSH_HOST=$(echo "$SSH_ENDPOINT" | sed 's|ssh://[^@]*@||' | cut -d: -f1)
          SSH_PORT=$(echo "$SSH_ENDPOINT" | cut -d: -f3)
          ssh -i .ssh/id_ed25519 \
            -p "$SSH_PORT" \
            -o StrictHostKeyChecking=no \
            -o UserKnownHostsFile=/dev/null \
            -o ServerAliveInterval=30 \
            -o ServerAliveCountMax=60 \
            "ci@$SSH_HOST" \
            "curl -f 'https://replicated.app/embedded/playball-exe/${CHANNEL}/${VERSION}' \
               -H 'Authorization: ${LICENSE_ID}' -o playball-exe.tgz && \
             tar xzf playball-exe.tgz && \
             sudo ./playball-exe install \
               --license license.yaml \
               --headless \
               --yes \
               --installer-password '${INSTALLER_PASSWORD}' \
               --config-values /tmp/config-values.yaml"

      - name: Archive customer
        if: always() && steps.create-customer.outcome != 'skipped'
        uses: replicatedhq/replicated-actions/archive-customer@v1
        with:
          api-token: ${{ secrets.REPLICATED_API_TOKEN }}
          customer-id: ${{ steps.create-customer.outputs.customer-id }}

      - name: Delete VM
        if: always() && steps.create-vm.outcome != 'skipped'
        run: |
          replicated vm rm "${{ steps.create-vm.outputs.vm-id }}" \
            --token ${{ secrets.REPLICATED_API_TOKEN }}
```

name: E2E Test on CMX

on:
  workflow_call:
    inputs:

        description: Pull request number
        required: false
        type: string
        default: 'manual'
      channel-slug:
        description: Replicated channel slug
        required: true
        type: string
      version:
        description: Release version label
        required: true
        type: string

jobs:
  e2e:
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - uses: actions/checkout@v4

      - name: Create test customer
        id: create-customer
        uses: replicatedhq/replicated-actions/create-customer@v1
        with:
          app-slug: playball-exe
          api-token: ${{ secrets.REPLICATED_API_TOKEN }}
          customer-name: ci-pr-${{ inputs.pr-number }}-${{ github.run_number }}
          license-type: dev
          channel-slug: ${{ inputs.channel-slug }}
          is-kots-install-enabled: "false"

      - name: Create CMX cluster
        id: create-cluster
        uses: replicatedhq/replicated-actions/create-cluster@v1
        with:
          api-token: ${{ secrets.REPLICATED_API_TOKEN }}
          kubernetes-distribution: k3s
          kubernetes-version: "1.32"
          cluster-name: ci-e2e-${{ github.run_id }}
          ttl: 1h
          export-kubeconfig: "true"

      - name: Generate nextauth secret
        run: echo "NEXTAUTH_SECRET=$(openssl rand -base64 32)" >> $GITHUB_ENV

      - name: Install via Replicated registry
        uses: replicatedhq/replicated-actions/helm-install@v1
        with:
          helm-path: helm
          chart: oci://registry.replicated.com/playball-exe/${{ inputs.channel-slug }}/playball-exe
          version: ${{ inputs.version }}
          name: playball
          namespace: default
          kubeconfig: ${{ steps.create-cluster.outputs.cluster-kubeconfig }}
          registry-username: ${{ steps.create-customer.outputs.license-id }}
          registry-password: ${{ steps.create-customer.outputs.license-id }}
          values: |
            nextauth:
              secret: "${{ env.NEXTAUTH_SECRET }}"
            service:
              type: NodePort
            image:
              tag: ${{ inputs.image-tag }}

      - name: Get NodePort
        id: get-nodeport
        run: echo "port=$(kubectl get svc playball -o jsonpath='{.spec.ports[0].nodePort}')" >> $GITHUB_OUTPUT

      - name: Expose app
        id: expose-port
        uses: replicatedhq/replicated-actions/expose-port@v1
        with:
          api-token: ${{ secrets.REPLICATED_API_TOKEN }}
          cluster-id: ${{ steps.create-cluster.outputs.cluster-id }}
          port: ${{ steps.get-nodeport.outputs.port }}
          protocols: http,https

      - name: Wait for app to be reachable
        run: |
          APP_URL="https://${{ steps.expose-port.outputs.hostname }}"
          until curl -sf --max-time 5 "$APP_URL" > /dev/null 2>&1; do
            echo "Waiting for app..."
            sleep 10
          done

      - name: Test registration
        run: |
          APP_URL="https://${{ steps.expose-port.outputs.hostname }}"
          RESPONSE=$(curl -sf -X POST "$APP_URL/api/auth/register" \
            -H "Content-Type: application/json" \
            -d '{"email":"ci@example.com","password":"cipassword123","franchiseName":"CI Franchise"}')
          echo "$RESPONSE"
          echo "$RESPONSE" | jq -e '.success == true'

      - name: Install support-bundle CLI
        id: install-support-bundle
        if: always() && steps.create-cluster.outcome != 'skipped'
        run: |
          curl -fsSL https://github.com/replicatedhq/troubleshoot/releases/latest/download/support-bundle_linux_amd64.tar.gz \
            | tar xz -C /tmp support-bundle
          sudo mv /tmp/support-bundle /usr/local/bin/support-bundle

      - name: Collect support bundle
        id: collect-support-bundle
        if: always() && steps.create-cluster.outcome != 'skipped' && steps.install-support-bundle.outcome == 'success'
        run: |
          support-bundle --load-cluster-specs \
            --output /tmp/support-bundle \
            --interactive=false

      - name: Upload support bundle
        if: always() && steps.create-cluster.outcome != 'skipped' && steps.collect-support-bundle.outcome == 'success'
        uses: actions/upload-artifact@v4
        with:
          name: support-bundle-${{ github.run_id }}
          path: /tmp/support-bundle.tar.gz
          retention-days: 7

      - name: Archive customer
        if: always() && steps.create-customer.outcome != 'skipped'
        uses: replicatedhq/replicated-actions/archive-customer@v1
        with:
          api-token: ${{ secrets.REPLICATED_API_TOKEN }}
          customer-id: ${{ steps.create-customer.outputs.customer-id }}

      - name: Delete cluster
        if: always() && steps.create-cluster.outcome != 'skipped'
        uses: replicatedhq/replicated-actions/remove-cluster@v1
        with:
          api-token: ${{ secrets.REPLICATED_API_TOKEN }}
          cluster-id: ${{ steps.create-cluster.outputs.cluster-id }}
```

name: Build and Test

on:
  pull_request:

jobs:
  build:
    uses: ./.github/workflows/docker-publish.yml
    with:
      tag: pr-${{ github.event.pull_request.number }}
    secrets: inherit

  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - run: npm ci

      - run: npm test

      - name: Helm lint
        run: helm lint deploy/charts --set nextauth.secret=test

      - name: Install Replicated CLI
        run: |
          version=$(curl -s https://api.github.com/repos/replicatedhq/replicated/releases/latest \
            | grep -m1 -Po '"tag_name":\s*"v\K[^"]+')
          curl -Ls "https://github.com/replicatedhq/replicated/releases/download/v${version}/replicated_${version}_linux_amd64.tar.gz" \
            -o replicated.tar.gz
          tar xf replicated.tar.gz replicated && rm replicated.tar.gz
          sudo mv replicated /usr/local/bin/replicated

      - name: Replicated release lint
        run: |
          helm package deploy/charts -d /tmp
          replicated release lint --chart /tmp/playball-exe-*.tgz
        env:
          REPLICATED_API_TOKEN: ${{ secrets.REPLICATED_API_TOKEN }}
          REPLICATED_APP: playball-exe

  release:
    needs: [build, lint]
    runs-on: ubuntu-latest
    outputs:
      channel-slug: ${{ steps.create-release.outputs.channel-slug }}
      version: ${{ steps.set-version.outputs.version }}
    steps:
      - uses: actions/checkout@v4

      - name: Set version
        id: set-version
        run: echo "version=0.1.0-pr.${{ github.event.pull_request.number }}.${{ github.run_number }}" >> $GITHUB_OUTPUT

      - name: Package Helm chart
        run: helm package deploy/charts -d deploy/manifests --version "${{ steps.set-version.outputs.version }}"

      - name: Install preflight CLI
        run: |
          curl -fsSL https://github.com/replicatedhq/troubleshoot/releases/latest/download/preflight_linux_amd64.tar.gz \
            | tar xz -C /tmp preflight
          sudo mv /tmp/preflight /usr/local/bin/preflight

      - name: Run preflight checks
        run: |
          helm template deploy/manifests/playball-exe-${{ steps.set-version.outputs.version }}.tgz \
            --set nextauth.secret=test \
            | preflight --interactive=false -

      - name: Create and promote Replicated release
        id: create-release
        uses: replicatedhq/replicated-actions/create-release@v1
        with:
          app-slug: playball-exe
          api-token: ${{ secrets.REPLICATED_API_TOKEN }}
          yaml-dir: deploy/manifests
          promote-channel: pr-${{ github.event.pull_request.number }}
          version: ${{ steps.set-version.outputs.version }}

  e2e:
    needs: [release]
    uses: ./.github/workflows/e2e.yml
    with:
      image-tag: pr-${{ github.event.pull_request.number }}
      pr-number: ${{ github.event.pull_request.number }}
      channel-slug: ${{ needs.release.outputs.channel-slug }}
      version: ${{ needs.release.outputs.version }}
    secrets: inherit

  e2e-ec3:
    needs: [release]
    uses: ./.github/workflows/e2e-ec3.yml
    with:
      image-tag: pr-${{ github.event.pull_request.number }}
      pr-number: ${{ github.event.pull_request.number }}
      channel-slug: ${{ needs.release.outputs.channel-slug }}
      version: ${{ needs.release.outputs.version }}
    secrets: inherit

  archive-channel:
    needs: [release, e2e, e2e-ec3]
    if: always() && needs.release.result != 'skipped'
    runs-on: ubuntu-latest
    steps:
      - uses: replicatedhq/replicated-actions/archive-channel@v1
        with:
          app-slug: playball-exe
          api-token: ${{ secrets.REPLICATED_API_TOKEN }}
          channel-slug: ${{ needs.release.outputs.channel-slug }}
```
